# Candidate Dashboard – Remote Staffing System

##  Project Overview
This project presents a *Candidate Dashboard built using Microsoft Power BI* to analyze candidate data, understand role preferences, and explore candidate distribution patterns.  
The dashboard is designed to support data-driven insights into candidate behavior and onboarding trends.

It demonstrates:
- KPI creation  
- Interactive dashboard design  
- Power BI visualization best practices  
- Analytical storytelling using candidate data  

---

##  Dashboard Features

The Candidate Dashboard provides the following insights:

###  Candidate Overview Metrics
- *Total Candidates* – Overall number of candidates in the system  
- *Candidates Onboarded Over Time* – Growth trend of new registrations  
- *Active / Valid Candidates* – Candidates with valid profile information  

###  Role-Based Insights
- *Candidates by Role* – Distribution of candidates across different roles  
- *Candidates by Requested Role* – Demand trend based on candidate preferences  

###  Distribution Insights
- *Candidates by Location* (if applied in visual level filters)  
- *Candidate trend by onboarding date*  

###  Interactivity
- Interactive visuals that allow filtering and exploration  
- Role-based and date-based analysis  
- User-friendly layout for easy interpretation  

---

##  Data Source Used

### CANDIDATES_FACT
The dashboard is built using a candidate dataset containing:
- CANDIDATE_ID  
- FULL_NAME  
- EMAIL  
- ROLE  
- REQUESTED_ROLE  
- CREATED_AT  
- INGESTION_DATE
